캐시 관리

캐시를 초기화하면 모든 저장된 손상 대책방안이 삭제되고, 다음 요청 시 다시 생성됩니다.

캐시 시스템 정보
캐시 시스템의 장점:
  • 속도 향상: 한 번 생성된 손상 대책방안은 캐시에서 즉시 조회
  • API 비용 절약: GPT API 호출 횟수 대폭 감소
  • 일관성 보장: 동일한 손상 유형에 대해 항상 같은 대책방안 제공
캐시 작동 방식:
  1. 손상 유형 + 부재명 + 보수방안 조합으로 고유 키 생성
  2. 캐시에 해당 조합이 있으면 즉시 반환
  3. 없으면 RAG 시스템으로 생성 후 캐시에 저장
